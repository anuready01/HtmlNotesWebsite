HTML Lists and Tables

Lists in HTML organize information into a structured, readable format, while tables arrange data in a grid of rows and columns. Using the right structure semantically improves accessibility for screen readers and makes your content easier for search engines to understand.

HTML Lists

1. Unordered lists <ul>

Unordered lists display a list of items where the order does not matter.

  • Starts with the <ul> tag.
  • Each list item begins with the <li> tag.
  • The default marker is a filled-in circle (a disc), but you can change it to a hollow circle (circle), a square (square), or to none with CSS.

Example:


<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Oranges</li>
</ul>

2. Ordered lists <ol>

Ordered lists are used for items where the sequence is important.

  • Starts with the <ol> tag.
  • Each list item begins with the <li> tag.
  • By default, items are marked with numbers, but you can use the type attribute to change it to uppercase letters (A), lowercase letters (a), uppercase Roman numerals (I), or lowercase Roman numerals (i).
  • The start attribute can be used to begin the list from a specific number.

Example with attributes:


<ol type="a" start="3">
  <li>Start the engine</li>
  <li>Check the mirrors</li>
  <li>Drive away</li>
</ol>

3. Description lists <dl>

Also known as definition lists, these are useful for defining terms.

  • The entire list is contained within the <dl> tag.
  • The term being defined is specified with the <dt> tag.
  • The description or definition for the term is specified with the <dd> tag.

Example:


<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

4. Nested lists

You can place one list inside another to create a hierarchy.

Example:


<ul>
  <li>Main Item One</li>
  <li>Main Item Two
    <ol>
      <li>Sub-item 2.1</li>
      <li>Sub-item 2.2</li>
    </ol>
  </li>
  <li>Main Item Three</li>
</ul>

HTML Tables

Basic table structure

HTML tables are created with the <table> tag and structured with rows and data cells.

  • <table>: The container for the entire table.
  • <tr>: Defines a single table row.
  • <th>: Defines a table header cell. Text is typically bold and centered by default.
  • <td>: Defines a standard table data cell.
  • <caption>: Provides a title or description for the table.

Example:


<table>
  <caption>Monthly Sales Report</caption>
  <tr>
    <th>Month</th>
    <th>Sales</th>
  </tr>
  <tr>
    <td>January</td>
    <td>$5000</td>
  </tr>
</table>

Advanced table structure and features

For more complex or accessible tables, you can use additional tags to define different parts of the table.

  • <thead>: Groups the header content in a table.
  • <tbody>: Groups the main body content of the table.
  • <tfoot>: Groups the footer content, such as a summary row.
  • <colgroup> and <col>: Specify properties for a group of columns.

Cell spanning attributes

These attributes allow a single cell to occupy the space of multiple rows or columns.

  • colspan: An attribute used on a <td> or <th> tag to specify how many columns a cell should span.
  • rowspan: An attribute used on a <td> or <th> tag to specify how many rows a cell should span.

Example with spanning cells:


<table>
  <tr>
    <th>Name</th>
    <th colspan="2">Contact Information</th>
  </tr>
  <tr>
    <td>Jane Doe</td>
    <td>555-1234</td>
    <td>jane.doe@email.com</td>
  </tr>
</table>

Styling tables with CSS

For modern web development, styling should be handled with CSS rather than using deprecated HTML attributes like bgcolor.

Common CSS properties for tables:

  • border: Add borders to the table and cells.
  • border-collapse: Collapse the cell borders into a single border.
  • padding: Control the space between the cell content and its border.
  • text-align: Align the content inside a cell.
  • background-color: Set a background color for the table, a row, or a specific cell.
